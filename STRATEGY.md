> **한 줄 요약:** 바쁜 현대인이 기사를 찾아다닐 필요 없이, 매일 **채팅방에서 종목별 요약 카드뉴스**를 받아보게 한다. 더 궁금하면 블로그에서 자세히 보고, **구독하면** 경기 전 인텔리전스 카드·개인화 팔로우를 이메일·텔레그램으로 받는다. 이 과정에서 **이메일·관심사 데이터**를 모으고, 트래픽이 쌓이면 **온라인 배팅 사이트 배너 광고**로 직접 수익화한다.

## 1. 문제와 기회

**문제 — 콘텐츠 사업의 구조적 병목**
- 사람이 쓰는 스포츠 콘텐츠는 **비싸고 느리고 일관성이 없다.** 매일 다국어로, 정해진 시간에, 품질을 유지하며 발행하는 것은 인건비로 감당이 안 된다.
- 단순 "뉴스 자동 생성"은 이미 commodity다. 그리고 그 길에는 세 개의 지뢰가 있다:
  - **표절** — 남의 기사를 다시 쓰면 저작권·"hot news" 침해
  - **환각** — AI가 자신만만하게 틀린 사실(스코어·이적료)을 만들어냄
  - **중복** — 같은 사건을 매일/여러 번 발행

**기회**
- 위 세 지뢰를 **자동으로, 결정론적으로** 막을 수 있다면, "무인 다국어 콘텐츠 공장"이 성립한다.
- 게다가 뉴스사이트는 *벌어진 일*만 보도하고 **의미를 패키징하지 않는다.** 여기에 빈 시장이 있다.

> **포지셔닝:** "뉴스를 모아주는 곳"이 아니라 **"바쁜 사람이 채팅방에서 스포츠를 다 따라잡게 해주는, 의미까지 주는 스포츠 콘텐츠 채널."** 핵심 타깃은 **기사를 일일이 찾아볼 시간이 없는 현대인.**

## 2. 우리의 해법 — 안전한 무인 발행

핵심은 두 개의 설계 원칙이다.

**① 다중 출처 합성 (Multi-source Synthesis)**
- 한 기사를 베끼지 않는다. 같은 사건을 다룬 **2~3개 출처의 *사실*만** 종합해, **우리 문체의 새 기사**를 쓴다.
- 변형적(transformative)이라 법적으로 방어 가능하고, 부수적으로 사실 교차검증·중복 병합 효과까지 얻는다.
- **출처 1개뿐이면 발행하지 않는다** — 단일 출처 재작성은 절대 금지(저작권 1차 방어선).

**② Fail-closed 게이트 (의심스러우면 발행하지 않는다)**
- 모든 단계가 "통과 못 하면 막는다"로 동작한다. 틀린 사실·깨진 이미지·중복을 내보내느니 **그날 그 건을 건너뛴다.**
- 모든 폴백은 **반드시 알림**을 남긴다 — 조용한 실패가 없으므로 사람 없이도 안전하다.

> **가장 중요한 통찰:** 사실 검증은 "AI에게 맞냐고 물어보기"가 **아니다.** AI에게 물으면 자신만만한 환각을 통과시킨다. 그래서 우리는 기사 속 숫자·이름·인용을 **출처 원문에 직접 대조**하는 결정론적 방식을 쓴다. (예: 스코어 `3-0`은 출처의 `13-0` 안에 있어도 매칭 안 됨 — 정수 전체일치)

## 3. 차별점 — 왜 우리 채팅방에 머무는가

"뉴스 모아주기"만으로는 약하다. 그래서 뉴스가 *못 하는* 두 가지를 얹었다.

### ① 경기 전 인텔리전스 카드
- **카드 1장** = 매치업 + 대회/킥오프 + 양 팀 최근 폼(W/D/L 배지) + 핵심 부상 + **"관전 포인트."**
- 뉴스는 '벌어진 일'을 보도하지만, 우리는 **"다음 경기에서 뭘 볼지, 무슨 의미인지"** 를 한 장으로 패키징.
- **고가치 트래픽 → 배너 단가 상승**으로 직결.
- *합성 노트도 사실 게이트를 통과해야 한다* — 근거 없는 부상/선수 주장은 카드 자체가 차단됨(뉴스와 동일한 규율).

### ② 개인화 팔로우 (구독 회원 기능)
- 봇에 `/follow 토트넘` → 그 팀/선수 콘텐츠만 골라 **개별 푸시.**
- 구독 회원은 본인 관심 정보를 **이메일 + 텔레그램**으로 바로 받아본다.
- 전체 방송(firehose)을 **"내 피드"** 로 바꿔, "왜 계속 이 방에 있나"의 답을 만든다 = **리텐션 엔진.**
- 커맨드: `/follow`, `/unfollow`, `/myfeed`. 뉴스 카드·프리매치 카드 둘 다에 적용.
- **부수 효과 = 데이터 자산:** 구독 과정에서 **이메일 + 관심사(팔로우 종목/팀/선수)** 가 쌓인다 → 재마케팅·타깃 광고의 기반.

> **요지:** ①은 **획득(고가치 트래픽)**, ②는 **리텐션 + 데이터 수집.** 둘 다 commodity 뉴스가 흉내 낼 수 없는 부분. (수익화 퍼널은 §7 참조)

## 4. 파이프라인 한눈에

```
매일 00:00 UTC — GitHub Actions가 자동으로 깨어남
        │
  [뉴스 사이트들] ──(수집)──▶ [이 프로젝트 / Node.js] ──(발행)──▶ [WordPress 블로그]
        │                          모으고→합치고→검증→카드제작              +
        │                                                            [Telegram 카드뉴스]
        ▼
   독자에게 도달 (브랜드 중립 · 순수 스포츠 정보)
```

**11단계 자동 흐름:** discover(수집) → cluster(같은 사건 병합) → dedup-A(기발행 제거) → synthesize(다중 출처 합성) → **사실 게이트** → **표절 게이트** → 카드 렌더 → glyph-smoke(글자깨짐 검사) → WordPress 발행(KO+EN) → Telegram 배포 → 원장 기록(다음날 중복 방지).

| 요소 | 역할 | 비용 |
|------|------|------|
| Node.js + GitHub Actions | 실행 엔진 + 매일 자동 타이머 + 시크릿 보관 | 무료(Public 레포) |
| Serper / NewsAPI | "오늘 뭐가 이슈인지" 발견 | 무료 티어 |
| Playwright + robots.txt | 허용된 페이지 본문만 수집 | 무료 |
| **Anthropic (Claude)** | 다중 출처 합성 + 사실 추출 | 글당 과금 |
| sharp | 카드뉴스 이미지 렌더(로컬) | 무료 |
| WordPress + Polylang + Rank Math | 발행 / 한·영 번역쌍 / SEO | 호스팅비 |
| Telegram Bot | 카드뉴스 배포 + 운영 알림 | 무료 |

## 5. 다섯 가지 약속 (매일 자동 검증)

"그냥 돌아간다"가 아니라, **매일 다음 5가지가 충족돼야 성공**으로 본다. 각각 자동 테스트로 검증.

| # | 약속 | 보장 방식 |
|---|------|----------|
| **AC#1** | 한국어+영어 기사가 자동 생성·발행 | WP에 KO·EN 발행 + Polylang 번역쌍 연결 |
| **AC#2** | 표절도 없고 사실 오류도 없다 | 표절 게이트(유사도) + 사실 게이트(출처 대조) |
| **AC#3** | 카드뉴스가 가되 깨진 이미지는 안 간다 | 렌더 후 글자 픽셀 검사(glyph-smoke) 통과 시에만 전송 |
| **AC#4** | 같은 기사를 두 번 발행하지 않는다 | 사건 지문 + 발행 원장으로 중복 제거 |
| **AC#5** | 사람 손 없이 매일 끝까지 돈다 | cron + 모든 실패가 "조용히 넘어가지 않고" 알림 |

> **녹색 체크의 의미:** 테스트는 "가짜 데이터가 우연히 피한 케이스"가 아니라 **"게이트가 잡도록 설계된 케이스"를 실제로 잡는다.** (독립 코드 리뷰에서 사실 게이트 오탐 3건 발견·수정 후 회귀 테스트로 고정)

## 6. 법적 방어 — 위험 등급별로 본다

> **핵심 메시지:** "우리가 직접 생성하니 다 안전하다"는 **절반만 맞다.** 저작권은 *사실*이 아니라 *표현*을 보호하므로, 사실만 뽑아 우리 문체로 쓰면 텍스트는 방어 가능하다 — **단, 그것을 가능케 하는 장치(다중 출처 합성·단일출처 폐기)가 있어야 한다.** 그 장치가 빠지면 "AI로 살짝 바꾼 표절"이 된다. 그리고 **수집(접근) 행위**와 **이미지**는 텍스트 생성과 완전히 별개의 축이다.

**결론 한 줄:** **텍스트는 구조로 방어 완료(위험 낮음) → 실질 리스크는 이미지에 집중 → 이미지 출처 게이트로 대응.**

| 위험 축 | 등급 | "직접 생성"으로 해결? | 대응 장치 |
|--------|:---:|:---:|------|
| **표현 복제** (텍스트) | 🟢 낮음(조건부) | △ | 사실만 추출+우리 표현. 단, 한 기사 근접 의역은 침해 → **단일 출처 폐기**가 선을 그음 |
| **Hot-news** (시사정보 무단이용) | 🟢 낮음 | ❌ | 순수 사실도 취재 무임승차는 별개 책임 → **다중 출처 합성**이 1차 방어 |
| **수집(접근) 행위** | 🟢 낮음 | ❌ | 출력물이 원본이어도 *가져오는 행위*가 약관위반/무단접근일 수 있음 → **robots.txt 강제 + ToS 검토 기록** |
| **선의·공정이용 태세** | — | ❌ | 분쟁 시 방어 태세 → **출처 표기 + 면책 페이지** |
| **이미지** (사진 저작권 + 선수 초상권) | 🔴 **높음·명확** | ❌ | 사진은 *표현 그 자체*라 합성 불가, 워터마크·역검색·DMCA로 적발 쉬움 → **이미지 출처 게이트:** 화이트리스트(공식 임베드/라이선스/Wikimedia)만, 없으면 **생성 이미지로 폴백.** 검색·다운로드·핫링크 재호스팅 금지 |

> **왜 이 4개(합성·폐기·robots·표기)를 빼면 안 되나:** 각각이 *다른* 법적 축을 덮는다. 다중출처 합성·단일출처 폐기는 "우리가 생성한다"는 말을 *진짜로 만들어주는* 장치이고, robots/ToS는 아예 다른 종류의 위험(접근 행위)이라 텍스트와 무관하게 따로 존재한다.

## 7. 비즈니스 모델

**핵심 타깃:** 기사를 일일이 찾아볼 시간이 없는 **바쁜 현대인.** 그들이 원하는 것은 "검색"이 아니라 **"채팅방에 앉아서 오늘 스포츠를 다 따라잡는 것."**

**가치 사다리 (Value Ladder) — 무료에서 구독, 그리고 데이터·수익으로**

```
[1] 무료 — 채팅방 종목별 요약 카드뉴스
    매일 텔레그램 채팅방에서 종목별 스포츠 요약 카드뉴스를 받아본다.
    "기사를 찾아보지 않아도 오늘 스포츠가 정리된다."
        │  더 궁금하면 ↓
[2] 무료 — 블로그 상세 기사 (트래픽 발생 지점)
    카드에서 더 알고 싶으면 블로그에서 KO/EN 상세 기사 확인 → 블로그 트래픽 축적
        │  더 깊이 원하면 ↓
[3] 구독 — 프리미엄 기능 + 개인 채널 배달
    ① 경기 전 인텔리전스 카드  ② 개인화 팔로우(내 팀/선수만)
    → 본인 관심 정보를 이메일 + 텔레그램으로 바로 받아봄
        │  그 과정에서 우리가 얻는 것 ↓
[4] 데이터 자산 — 이메일 + 관심사(팔로우 종목/팀/선수) 수집
        │  이용자·트래픽이 임계점을 넘으면 ↓
[5] 수익화 — 온라인 배팅 사이트 배너 광고 (직접 수익)
```

**왜 이 구조가 작동하는가**
- **진입 장벽 0:** 채팅방 카드뉴스는 가입·검색 없이 즉시 가치 제공 → 빠른 유입.
- **자연스러운 트래픽 펌프:** "자세히"가 궁금해지는 순간이 블로그 방문으로 전환 → SEO 자산 + 광고 노출 면.
- **구독으로 데이터 전환:** 프리미엄(①②) + 이메일/텔레그램 배달을 미끼로 **이메일·관심사**를 확보 → 재마케팅·타깃팅 자산.
- **수익은 트래픽의 함수:** 이용자·트래픽이 쌓이면 **온라인 배팅 사이트 배너 광고**로 직접 수익. 트래픽 *품질*(① 인텔 카드로 끌어온 고관여 스포츠 독자)이 배너 단가를 끌어올림 — 배팅 광고주에게 가장 가치 있는 오디언스이기 때문.

> **데이터 = 두 번째 제품.** 우리는 콘텐츠를 팔지 않는다. **고관여 스포츠 오디언스(이메일·관심사 포함)를 모아 배팅 광고주에게 도달 면을 판다.** 콘텐츠는 그 오디언스를 모으는 무료 미끼다.

**구현 단계 매핑**
- 현재 MVP(✅): [1] 채팅방 카드뉴스 + [2] 블로그 + [3]의 ①②(텔레그램 배달).
- 라이브 후 단기: 이메일 채널 + 구독자 데이터 스토어(이메일·관심사) 배선.
- 트래픽 임계 후: [5] 배팅 배너 광고 도입.

**비용 구조 (글 1건 기준)**

| 항목 | 비용 |
|------|------|
| Claude Sonnet 4.6 (합성 KO+EN 번역) | 분량 비례 |
| Claude Sonnet 4.6 (사실 추출, temp 0) | 소액 |
| 자문 평가 (Haiku + GPT-4o mini + Gemini Flash, *차단 권한 없음*) | 소액/무료 |
| sharp 카드뉴스 · Serper/NewsAPI | $0 / 무료 티어 |

> **예산 틀:** 브랜드당 월 $10K 예산 내. 정확한 일일 상한은 **라이브 후 측정**해 종목 확장 전에 확정. (비용 통제 위해 합성은 opus가 아닌 sonnet-4-6 채택)

   → 실제 출처 교체 → GitHub cron 활성화(dry-run green일 때만)
        │
        ▼
[Phase 3] 수익화 + 확장 🟢 (검증 후)
   이메일 채널 + 구독자 데이터 스토어(이메일·관심사) 배선

## 부록 B — 데모 (시크릿 0개로 즉석 시연 가능)

```bash
git clone <repo> && cd sports-news-blog-project
npm install                 # 런타임 의존성은 sharp 하나뿐
npm run pipeline:dry-run    # 전체 파이프라인을 가짜 데이터로 실행 + 5개 AC 자동 검증
```
기대 출력:
```
Discovered 5 items -> 3 events
Published: 1 | Fact-blocked: 1 | Follower-pushes: 1
Pre-match: built 1 | fact-blocked 1 | follower-pushes 1
  [PASS] AC#1 ... [PASS] AC#5
ALL CHECKS PASS (5 ACs + ① pre-match + ② personalization)
```
> 의미: 5개 원재료 → 3개 사건 병합 → 1건 발행, 1건 **사실 차단(3-0 오류)**, 1건 부적격(출처 1개), 2회차 중복제거 0건. → **게이트가 실제로 작동함을 청중 앞에서 증명.**

**Anthropic 키만으로 실제 Claude 시연:**
```bash
ANTHROPIC_API_KEY=sk-ant-... npm run pipeline:run
# 실제 Claude가 합성한 기사가 실제 사실 게이트를 통과/차단
```
